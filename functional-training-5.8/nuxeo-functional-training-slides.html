##Installation et démarrage
###Téléchargement Disponible depuis nuxeo.com : http://www.nuxeo.com/fr/telechargez-nuxeo

Packages disponibles: - Package multi-plateforme (.zip) : Serveur Nuxeo complet zippé - Installeur Windows (.exe) : installe les dépendances si nécessaire (Java, OpenOffice, etc.) - Installeur Debian/Linux : installe les dépendances et un script init - Images de machine virtuelles

Toutes les procédures d'installation : doc.nuxeo.com

###Installation

Prérequis : Java 7

Où : Racine d'un disque (C:\ par exemple)

- Pas de problème de profondeur de chemin - Pas de problème de chemin avec des espaces / caractères spéciaux

Quoi : Serveur Nuxeo

###Démarrage

Double-clic sur Start Nuxeo.bat

-> Panel Control pour arrêter le serveur

URL: http://localhost:8080/nuxeo

###Assistant de configuration

Premier démarrage du serveur : assistant de configuration pour vous aider à régler les paramètres de configuration les plus courants.

Redémarrage à la fin de l'assistant

###Paramètres de configuration - Base de données : h2 (base de données embarquée)
Tests, démo, développement uniquement - SMTP : pour l'envoi de notifications par email aux utilisateurs
Facultatif - Proxy - Connect : Accès à la Marketplace et Nuxeo Studio - Modules : Document Management

##Droits d'accès
###Droits d'accès ####Définition

Ensemble de permissions qui détermine les actions disponibles. Les droits d'accès peuvent être accordés ou refusés.

####Pour qui

Déterminés pour :

- utilisateur individuel - groupe d'utilisateur ####Où ça

Appliqués sur les documents “folderish” = pouvant contenir d’autres documents

- Domaine - Espace de travail, de publication - Dossier - ...
###Héritage des droits

Droits hérités dans les sous-espaces

Blocage de l’héritage = réinitialisation des droits d’accès

-> Seul l’utilisateur ayant bloqué l’héritage a accès à l’espace (+ adminisrators)

###Les différents droits d'accès - Lecture - Ecriture - Suppression - Administration (tout gérer)
###Lecture - Consultation du contenu - Ajout de commentaires - Notifications - Envoi d’emails
###Ecriture - Création et modification de contenu - Suppression de contenu - Workflows - Relations - Publication - + Lecture
###Administration (Gérer tout) - Gestion des droits d’accès - Configuration locale - Administration de la publication - Corbeille - + Ecriture - + Lecture
###Interface de gestion des droits
    - Droits mis en place dans un espace parent - Droits mis en place dans l’espace courant - Formulaire d’ajout de droits d’accès dans l’espace courant
###Exercice

Donnez des droits sur les espaces de travail selon leur service :

- Les clients ne doivent pas avoir accès aux espaces de travail. - Les chefs de projets sont responsables de sections.

###Workflow en série

Ordre d'intervention des participants

Simple revue ou validation

- Pas de changement d'état - Projet -> Valide
###Workflow en parallèle

Objectif : valider ou non un document après une étape de consultation auprès de plusieurs participants

Parallèle car pas d'ordre d'intervention des participants

Déroulement :

    - Initiateur demande leur avis à des participants - Les participants indiquent : OK, pas OK, sans opinion - L'initiateur a le résumé de la consultation et décide de valider ou non le document.
