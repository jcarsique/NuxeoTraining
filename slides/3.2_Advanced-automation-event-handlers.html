###«Fn» Object Collection of useful helpers
available in automation parameters when using the script (« @{ script_expression}»)
- getNextId(string)
returns the next sequence item for the given string (enables to maintain in parallel multiple sequences)
- getDate()
return a Date object from a date property
- getVocabularyLabel(VocabularyName, KeyName)
returns the label for a vocabulary entry. (ex: useful when automatically updating some fields)
- getEmailsFromGroup(String groupName), getEmails...
useful helpers for resolving users, emails, etc, ...

##Nuxeo Training - Event handlers
###Event handlers #### At the end of the course: The publicationDate is pre-filled with the current date in the creation layout !
###Event ?

An Event is an action that is usually initiated outside the scope of a program and that is handled by a piece of code inside the program

Wikipedia

###Event Handlers - Listener pattern - When an event is raised (document created, document saved, etc.) - Add custom and dynamic behaviors without having to change the code. 
###Event Handlers - 2 types of event handlers - Synchronous - Same thread, coreSession and context - Post Commit asynchronous - Executed after a commit on the coreSession - New thread, new coreSession, new context - Isolated from each other - It will be executed asynchronously
###Event Handlers – why post commit async ? - Useful for external notification, when we want to be sure that the modifications are commited in the DB 
###Event Handlers with Nuxeo Studio - Select the events to listen to 
###Event Handler Enablement - Like we did with actions, select the event handler filter 
###Event Handlers Linked to an Operation Chain
###Document Events: creation - Empty document created - About to create - Document created Don't save the document if the document is not created
###Document Events: modification - Before document modification - Document modified Don't save the document if you deal with document modification events
