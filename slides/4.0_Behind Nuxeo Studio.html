##Extension points
###Extension points - Contribute configuration to existing EPs  - Contribute code and behavior to existing EPs  - Other components extend / customize your service
###Contribution to an extension point - The Extension point mecanism provide a way to extend a feature without changing any java code in the bundle that provides the feature.  - To contribute, In studio : - Advanced settings / XML extension  - just specify the component where the extension point is declare and its name  
    
<extension target="org.nuxeo.ecm.platform.actions.ActionService"
 point="actions">
   <action id="logout"
        link="#{loginLogoutAction.logout}"  
        label="command.logout">  
        <category>USER_SERVICES</category>
   </action>
</extension>

###XP Example : actions service
###Contribution to an extension point - Explore the existing extension points (and their contributions) with the explorer - Online : http://explorer.nuxeo.org - On your instance by installing the package from the market place - Let's have a look at studio with the generated contribution to extension points 
###Overriding existing contribution - 2 types of extension points :  - Can be overriden  - Can be merged - Contribution need to use the same id  - Override will completly replace the contribution object  - Merge will keep existing attributes if not overriden - Check the extension point documentation !
###Overriding existing contribution - Use the <require>componentToBeLoadedBefore</require> tag to make sure your contribution is registered after the original one.
