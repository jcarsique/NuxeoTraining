  • Parse phase → Component tree build phase → application phases (cf. JSF life cycle)→ Render phase 
  • Component tree build phase is executed for each Ajax render phase
  • A component (in the xhtml) can modify the tree component (Facelet, this is how layouts works)

JSF LifeCycle : 6 steps

  1. Build tree (Restore View), parses JSP/Facelets and creates component tree
  2. Apply request values, parses request and set component tree local values 
  3. Process validation, apply validators 
  4. Update model (EL value), update backing beans 
  5. Invoke Application (navigation/action), call the application logic 
  6. Render response (EL value) 

Each phase is done for the whole tree before the next phase

Overriding existing xhtml pages

  • Jsf xhtml page are contributed by several plugins
  • Pages are stored in the nuxeo.war folder
  • The deployment-fragment.xml is used to copy the ressources for each bundle
  • A require in deployment-fragment is needed to make sure the page to override is copied before. 
      
<?xml version="1.0"?>
<fragment version="1">
  <require>org.nuxeo.ecm.platform.tag.web</require>
  <install>
    <unzip from="${bundle.fileName}" to="/" prefix="web">
      <include>web/nuxeo.war/**</include>
    </unzip>
  </install>
</fragment>
