#Nuxeo Training - Content views, NXQL, Search forms
### Context

Now that they can organise their books collection, the "Best books" library would be interested in finer tuning of their display and search. Especially having the right information when browsing a book library, and a customized advanced search form.

###What we want to do ####At the end of the course: - A custom listing of books when browsing a book library - A custom advanced search form
###Content views #### Introduction

Any list of documents that is the result of a specific query

####Used in many places in Nuxeo: - for the content of a folderish node - for the trash folder - for advanced search - for simple search - for faceted search - for virtual navigation - for custom business content listing on some projects
###Content views #### Concepts - Divided into two tabs: - The Query and form tab defines which documents will be displayed - The Results tab defines how they should be displayed - Listed documents are the result of a query - Querying language used in Nuxeo is NXQL, a variant of SQL integrating specific options - Fields can be added to let the user filter the query results - A layout defines the columns displayed for the result
###NXQL - Nuxeo Query Language #### Concepts - Query language used to search documents, similar to SQL  - Only SELECT (no update or delete) - No JOIN - Can use: - System fields: `ecm:primaryType`, `ecm:currentLifeCycleSate` - Document metadata: `schema_prefix:property_name` Example: `SELECT * FROM Book WHERE bk:isbn LIKE '%12%'` #### Tip: test your query Use the advanced search as an administrator and go to the bottom of the screen. ####Full documentation http://doc.nuxeo.com/display/NXDOC56/NXQL
###NXQL - Examples 

SELECT * FROM Document WHERE ecm:mixinType = 'Versionable' AND ecm:mixinType <> 'Downloadable'

SELECT * FROM Document WHERE ecm:parentId = '5442fff5-06f1-47c9-ac59-1e10ef8e985b'

SELECT * FROM Document WHERE dc:coverage STARTSWITH 'foo/bar'

SELECT * FROM Document WHERE dc:title = 'hello world 1' ORDER BY ecm:currentLifeCycleState

SELECT * FROM Document WHERE ecm:fulltext = 'world'
### Query filter - Uses "where" clause of NXQL - Can use variables (EL expression language, giving access to Seam beans), ex: - `#{currentDocument.id}` - `#{currentDocument.path}` - `#{currentDocument.schema.property}` - `#{currentUser.name}` - Use `?` In the query filter and add query parameters in the same order to use variables - Example query 
ecm:mixinType != 'HiddenInNavigation' 
AND ecm:isCheckedInVersion = 0 
AND ecm:currentLifeCycleState != 'deleted' 
AND ecm:parentId = ?
- Parameters 
#{currentDocument.id}
###Result layout - Defines the result columns - Displays fields with layout widgets - Reusable: `Table Layout` (Listings & Views > Table Layouts)
### Practice #### Content views - Create a new content view `BookLibraryContentView` - Customize the query to get the children documents of the current book library. - Hint: if this is too difficult, go back a couple of pages… ;-) - Display the following columns in the result layout: - Doc type Icon, Title (with link), ISBN, Author, Creation date, Lifecycle state - Setup the `BookLibrary` document type to display this content view - Hint: there is only one main tab in the document types we haven't been through yet - Commit your changes - Deploy your changes - Have a look at a book library's content tab
### Custom advanced search form #### Concepts - Nuxeo studio provides easy access to advanced search form customization (Listing and views > Advanced search) - More advanced searches can be configured by creating content views with the "advanced search" flag - Specific advanced searches can be setup at Domain/Workspace level (manage tab, local configuration subtab)
### Practice #### Advanced Search - Create a custom search form - Search through the title (using ILIKE operator) - Search through the path (using STARTSWITH operator) - Search through creation time (using BETWEEN operator)
