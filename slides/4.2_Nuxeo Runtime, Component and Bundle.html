A Bundle?

A Bundle?

  • Every Nuxeo module is a bundle 
  • A bundle:
    • A jar file
    • Defined in the META-INF/MANIFEST.MF
    • Has a unique ID (bundle-symbolicName)
    • Declares a set of components 

Components

  • Defined in XML descriptors 
  • Declared in the MANIFEST bundle 
  • Provides: 
    • One or more services 
    • One or more Extension Points
    • One or more Extension Points xml contribution

Extension Points

  • Contribute configuration to existing EPs
  • Contribute code and behavior to existing EPs 
  • Other components extend / customize your service 

Extension Points

MANIFEST.MF

Contains OSGi entries:

        
Bundle-ManifestVersion: 1
Bundle-Name: your bundle name
Bundle-Version: 1.0.0
Bundle-Vendor: your company name

MANIFEST.MF

The unique name representing your bundle:

        
Bundle-SymbolicName: a.unique.name

Specific to Nuxeo: component registration:

        
Nuxeo-Component: OSGI-INF/one-component.xml, OSGI-INF/another-one.xml
  • Notes 
    • Be careful with spaces and commas
    • Pay attention to your SymbolicName - it has to be unique. A good choice is the package you are providing 
    • Always leave a new line at the end of the file 

Exercise - Your first bundle

Project Structure

  • With Nuxeo IDE, create a new Nuxeo Project. Choose a name, nuxeo-book-management can be a nice choice 
  •  It will automatically 
    • Creates folders
      • src/main/java and src/main/resources 
      • OSGI-INF and META-INF folders in resources 
    • Creates a MANIFEST.MF file in META-INF 
    • Bind the project to your SDK 
###Create a new automation gadget - Use the Nuxeo IDE wizard template - Need nuxeo-dm (use the right nxserver/data/installAfterRestart.log or install it from the Admin Center) - Deploy your changes on your sdk - Check that your automation gadget is available

Deploy Your Bundle (without the sdk)

  • Synchronize your pom (you may have new dependencies)
  • Build the jar with Maven
    • mvn clean package: package your project as a JAR 
    • mvn clean install: package your project and install it in your local Maven repository 
    • The jar file is in the working directory: target folder
    • Dependency issues ? See Nuxeo Maven Usage documentation
  • Put your JAR archive in the bundles folder on a standard Nuxeo Server (not sdk) 
  • (nxserver/bundles) for those who don’t remember 
  • Check that your automation gadget is available
