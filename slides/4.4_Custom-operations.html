Nuxeo Training - Custom operations

### Java operations #### What are they ? - Operations that you can call in an automation chain, for example in Nuxeo studio #### Why code Java operations ? - Lots of available operations are available in Nuxeo studio, but it may not be enough - Studio is suited for chains configuration, Java for advanced customization: - Access to services - Custom behaviour that can't be generic and integrated in Studio by Nuxeo - Missing operations in Studio that are not developed yet
### Creating a java operation #### Create the class - Use the NX wizard to create a new operation #### Annotate - The class with @Operation Â  - The runnable method with @OperationMethod - Operation parameters with @Param on attributes #### The runnable methods - Multiple operation methods can have the same name (with different signatures) - Inputs as method parameters - Output as returned object
### Operation sample 
@Operation(id = SampleOperation.ID, category = Constants.CAT_SERVICES, 
label = "Sample Operation", description = "An example.")

public class SampleOperation {
  public static final Log log = LogFactory.getLog(SampleOperation.class);
  public static final String ID = "Sample.Operation";

  @Context
  protected CoreSession session;

  @Context
  protected OperationContext ctx;
  
  @Param(name = "type")
  protected String type;

  @Param(name = "name", required = false)
  protected String name;
   
  @OperationMethod
  public DocumentModel run(DocumentModel doc) throws Exception {
    log.info("runing sample operation with params: type=" + type + "; name=" + name);
    return doc;
  }
}

### Register the operation - Registering the operation makes it available in Nuxeo platform - Use an XML extension in Nuxeo studio to register it 
<extension
  target="org.nuxeo.ecm.core.operation.OperationServiceComponent"
  point="operations">
  <operation class="org.nuxeo.example.TestOperation" />
</extension>
### Export the operation in Studio #### 2 methods to declare your operation in Studio ##### Automatically - Use the export button in the top left Nuxeo Studio tab and follow the wizard ##### Manually - Go to http://localhost:8080/nuxeo/site/automation/doc - Copy the JSON definition of the custom operation - Nuxeo Studio > Advanced settings > Registries > Operations - Paste the JSON definition surrounded with: 
{
 operations: [ ... ]
}
###Parameter widgets Widgets can be set programatically for operation parameters using annotations. 
@Param(
    name = "level",
    required = true,
    widget = Constants.W_OPTION,
    values = {"info", "debug", "warn", "error" })
protected String level = "info";

### Practice - Nuxeo IDE #### Custom operations (1/2) - Create a unit test named `shouldBeInTheBestBooksGroup()` that: - Runs with the `PlatformFeature` - Calls the `UserManager` service - Asserts that the user John and the BestBooks group are not null - Runs the `AddUserToGroup.run()` method with two parameters:
John as username and BestBooks as group - Fetches the user named `John` as a `NuxeoPrincipal` object - Asserts that John is a member of the BestBooks group The test will fail because the method does not exist yet. But now we know what it should do and will be able to implement it easily.
### Practice - Nuxeo IDE #### Custom operations (2/2) - Create a custom operation named `AddUserToGroup` - The operation will take two parameters: - `username` (required) - `group` (required, option list: people can choose between `BestBooks` and `Managers`) - The operation method should: - Call the `UserManager` service - Fetch the user as a `NuxeoPrincipal` object - Get the user's groups and add it to the selected group if not a member already Re run your test. If you did it correctly, your test should pass !
